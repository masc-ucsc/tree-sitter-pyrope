# tree-sitter-pyrope — agent guide

This repo holds the tree-sitter grammar for Pyrope plus the tooling built on it.
Read this before changing the grammar or the surrounding subsystems.

## Subsystems

- **Grammar** — `grammar.js` (hand-written) → `src/parser.c` (generated by
  `tree-sitter generate`). `src/scanner.c` is hand-written. `queries/*.scm` are
  the editor highlight/indent/fold queries.
- **Editor schemes** — `editors/nvim/pyrope.lua` + `queries/`. Both `queries/`
  and the grammar must stay at the repo root: `tree-sitter.json` and the
  nvim-treesitter `install_info` reference them by repo-relative path.
- **prpfmt** — `prpfmt/`, a C++20 formatter that links the generated parser
  directly (`*.cc` built as C++20; `parser.c`/`scanner.c` linked in as C).
  Self-contained; see `prpfmt/README.md`.
- **prpparse** — `prpparse/`, a hand-written recursive-descent Pyrope parser
  (bazel/C++23) producing an `hhds` tree, built to replace tree-sitter as the
  LiveHD front-end. Uses this grammar as a differential oracle (accepts exactly
  what `grammar.js` accepts; parses all 428 corpus files). Lexer + tree schema +
  parser are implemented (Phases 0–3); splitting/perf/LiveHD-integration are
  deferred. See `prpparse/README.md` and `prpparse/plan.md`.

## Build & test (all via the root Makefile)

```bash
npm install          # one-time: installs node_modules/tree-sitter-cli
make generate        # grammar.js -> src/parser.c
make test            # canonical check: grammar + prpfmt (test-grammar + test-prpfmt)
make test-all        # also prpparse (bazel unit tests + corpus accept-parity)
make corpus          # rebuild full_pyrope/ from ../docs (see below)
```

`make test` is the authoritative regression check (must stay green). It runs
`test-grammar` (`scripts/test.sh`, which parses every `full_pyrope/*.prp`) and
`test-prpfmt` (reformats every corpus file and re-parses the output). Run
`make test-grammar` alone for just the parse check.

Debug a single file:

```bash
./node_modules/tree-sitter-cli/tree-sitter parse -t ./full_pyrope/file1.prp
```

## The test corpus

`full_pyrope/*.prp` is **auto-generated and git-ignored** — never commit it. It
is extracted from the (non-deprecated) Pyrope docs:

```bash
make corpus
# equivalently:
./scripts/extract.rb -d ./full_pyrope/ ../docs/docs/pyrope/0*.md ../docs/docs/pyrope/1*.md
```

This requires the `../docs` checkout next to this repo. The `0*.md 1*.md` glob
deliberately excludes the `2*-deprecated*.md` files.

## Grammar development workflow

1. Edit `grammar.js`.
2. `make generate`.
3. `make test-grammar` and iterate until clean.
4. For a specific failure, inspect the tree with `tree-sitter parse -t <file>`.

## Documentation trust policy

⚠️ **DO NOT BLINDLY TRUST THE DOCS.** The corpus is extracted from
`../docs/docs/pyrope/*.md`, which contains typos, inconsistencies, and dubious
examples. When you hit syntax that looks wrong or "something I wouldn't write":

- **STOP** — don't add grammar complexity to accommodate it.
- **ASK** whether it's a real feature or a doc bug.
- Prefer fixing the source doc (then `make corpus`) over bending the grammar.

Common doc bugs: wrong comment syntax (`#` instead of `//`), bare standalone
expressions, inconsistent function syntax, missing assignments.

## Things to watch for in the grammar

- Precedence conflicts between rules.
- Missing or incorrect field names.
- The hand-written scanner (`src/scanner.c`) and its virtual-semicolon rule —
  `prpparse/plan.md` documents the exact terminator logic.
